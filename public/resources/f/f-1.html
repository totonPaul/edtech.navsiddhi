Web Application Development - Introduction

Web Applications over the internet!

Any Web Application has three components - Frontend, Backend and the Database.

Frontend of a web application is the GUI (Graphical user interface) with which you interact on websites, basically whatever you see on web pages is the frontend. The backend refers to the server-side of an application, where the logic, database interactions, and API integrations occur. It handles data processing, storage, and communication between the database and frontend, ensuring the application functions properly. Database is where all the data of the web application gets stored in a structured way.

In this course we will follow a framework MERN stack for building end to end web applications. MERN (MongoDB, Express.js, React, Node.js) is a JavaScript stack that simplifies integration by using a unified language (JS) across the backend, frontend, and database layers.

How does a web application work?

Frontend code, Backend code and database all three are present on the web server of a website.

Whenever you type a website's name on your browser, your browser fecthes the Frontend Code (suppose HTML, CSS, JS) from the web server of the website. The browser then converts the frontend code into an executable format that can be executed by the CPU of your device.

Whenever frontend wants to communicate with the backend or the database over the internet, it uses API ( Application programming Interface ) to do so.
