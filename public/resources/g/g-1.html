Android Application Development - Introduction

Android Applications over the internet!

Any Android Application has three components - Frontend, Backend and the Database.

Frontend of an android app is the GUI (Graphical user interface) with which you interact on android apps, basically whatever you see when you open an android app is the frontend of the app. The backend refers to the server-side of an application, where the logic, database interactions, and API integrations occur. It handles data processing, storage, and communication between the database and frontend, ensuring the application functions properly. Database is where all the data of the android application gets stored in a structured way.

We will use XML and JAVA for building android applications.

How does an android application work?

Frontend ( GUI code in XML and logic code in JAVA ) is compiled and uploaded to a platform like Google Playstore, from where end users can download, install and finally run the code on their devices.

For an online android application with internet support, its backend code and the database, both are present on a server. At a high level, a server can be understood as a computer where all the resources of the application are hosted.

Whenever frontend (code installed on your device) wants to communicate with the backend or the database over the internet, it uses API ( Application programming Interface ) to do so.
